# Question-Answering Log Analyzer

## Overview

The `analyze_qa_logs.py` tool provides a simplified, human-readable overview of how AI analysts perform their work during question-answering sessions. It parses log files generated by `question_answering.py` and extracts key information about each analyst's contributions.

## Features

For each AI analyst call, the tool displays:

- **Substantive unit being analyzed** (e.g., Section 5, Article 12)
- **Number of new facts** being added to the scratch document
- **Number of new questions** being asked
- **Specifics of questions** - full text of each question with importance ratings
- **Specifics of detail requests** - target articles and detail levels requested
- **Number of answers** provided to existing questions
- **Number of section references** added

## Usage

### Basic Usage

```bash
python analyze_qa_logs.py <logfile.json>
```

This displays an overview of all analyst calls, including those that took no actions. Scoring calls are always excluded (they are not analyst work, just relevance scoring).

### Verbose Mode

```bash
python analyze_qa_logs.py <logfile.json> --verbose
```

or

```bash
python analyze_qa_logs.py <logfile.json> -v
```

Verbose mode shows additional details including:
- Full text of facts added (truncated to 200 characters)
- Full text of answers provided (truncated to 200 characters)
- Section references added

### Show All Calls

```bash
python analyze_qa_logs.py <logfile.json> --show-all
```

or

```bash
python analyze_qa_logs.py <logfile.json> -a
```

Shows cleanup/answer generation passes that are normally hidden.

### Combining Options

```bash
python analyze_qa_logs.py <logfile.json> --verbose --show-all
```

## Understanding the Output

### Call Types

- **Analyst calls with actions**: Calls where the AI analyst added facts, questions, answers, or requests
- **Analyst calls with no actions**: Calls where the analyst reviewed but found nothing to add (often refinement passes)
- **Scoring calls**: Calls used to score relevance of units (always excluded from output)
- **Cleanup calls**: Calls that clean up and synthesize the scratch document (excluded unless --show-all is used)

### Action Types

- **Facts**: Information relevant to answering the primary question, extracted from the substantive unit
- **Questions**: Queries about OTHER substantive units (not the one being analyzed)
- **Answers**: Responses to questions asked by other analysts
- **Detail requests**: Requests for more detailed summaries (summary_2) of other units
- **Section references**: Indications that a question is also relevant to this unit

### Importance Ratings

Questions include importance ratings (1-3):
- **1**: Low importance
- **2**: Medium importance
- **3**: High importance (clearly important for answering the primary question)

## Tips

1. **Default mode shows all analyst calls** - Including those with no actions, which is important for understanding analyst behavior
2. **Scoring calls are always excluded** - They're not analyst work, just relevance scoring
3. **Use --verbose** when you want to see the full text of facts and answers that were added
4. **Look for patterns** in questions to understand what information the analysts need
5. **Check detail requests** to see which units need deeper analysis
6. **No-action calls are informative** - They show where analysts found nothing relevant to add

## Integration with Question-Answering Pipeline

This tool is designed to work with log files from `question_answering.py`. The typical workflow is:

1. Run `question_answering.py` on a document with a question
2. Review the generated log file with `analyze_qa_logs.py`
3. Understand how analysts approached the question
4. Identify areas where more detail was needed
5. Use insights to refine the question or approach

## Technical Notes

- The tool parses log entries written by `utils/ai_client.py`
- Each log entry is a JSON array: `[timestamp, full_cache, query_prompt, result, cache_created, cache_read]`
- The tool extracts substantive unit designations from the prompt text
- Action objects are parsed from the AI response JSON
- Calls are categorized based on their response patterns
